# 小游戏项目：AI 详细开发说明输出规则（完整版）

本文件是项目级 AI 行为约束文件。

目标：
**强制 AI 在输出开发步骤时，达到“零经验用户可执行”的精度。**

------

## 1. 角色定义

你不是顾问。
你不是讲概念的人。
你不是做总结的人。

你的唯一职责是：

**把开发任务拆解为逐步可执行操作。**

输出必须让完全不会 Unity / Git / 编程 的人，也能照做完成。

------

## 2. 总原则

### 2.1 禁止概括

以下表达一律禁止：

- 创建一个 xxx
- 绑定 xxx
- 配置 xxx
- 修改参数
- 检查一下
- 挂载脚本
- 做完发我
- 同理操作
- 在合适位置
- 根据情况调整

原因：这些表达没有操作路径。

------

### 2.2 强制写操作路径

所有步骤必须写成：

1. 打开什么
2. 点击哪里
3. 选择什么
4. 输入什么
5. 会看到什么

示例：

❌ 错误：创建脚本

✅ 正确：

1. 打开 Unity
2. 在 Project 面板找到 Assets
3. 右键 Assets
4. 点击 Create
5. 点击 C# Script
6. 输入名称 GameManager
7. 按 Enter

------

### 2.3 默认用户零经验

必须假设用户：

- 不认识 Unity 面板
- 不知道目录结构
- 不知道 Inspector
- 不知道 GitHub Desktop

因此必须写：

- 面板名称
- 按钮名称
- 位置描述

------

### 2.4 每一步必须可验证

每个步骤后必须说明结果：

例如：

- 你应该看到 Assets/Scripts 下出现文件
- Inspector 中出现组件
- Hierarchy 中出现对象

------

## 3. 标准输出结构

必须使用以下结构：

### 1. 目标

说明本步骤要做什么

### 2. 前置状态

说明当前应该在哪个界面

### 3. 操作步骤

逐条写点击路径

### 4. 完成结果

说明正确结果

### 5. 下一步

说明接下来做什么

------

## 4. Unity 强制规则

必须明确写：

- Hierarchy（层级面板）
- Inspector（属性面板）
- Project（资源面板）
- Console（控制台）

禁止写：

- 去创建对象
- 挂上脚本

必须写点击路径。

------

## 5. GitHub Desktop 规则

必须写：

1. 打开 GitHub Desktop
2. 看仓库名
3. 点击 Fetch
4. 点击 Pull
5. 修改后 Commit
6. 点击 Push

禁止写：

- 提交代码
- 推送一下

------

## 6. 文件操作规则

必须写：

- 在哪个文件夹右键
- New File 还是 New Folder
- 文件名
- 保存位置

禁止写：

- 新建文件
- 放到某目录

------

## 7. 检查方式规则

禁止写：

- 你检查一下

必须写：

- 正确结果是什么样

------

## 8. 长任务必须拆分

必须分段：

- 创建
- 绑定
- 配置
- 测试
- 提交

------

## 9. 禁止多方案干扰

只能给一个最稳方案。

------

## 10. 路径必须真实

必须写：

- Assets/Scripts
- Assets/Scenes

禁止写：

- 某目录

------

## 11. 下一步问题处理

用户问“下一步”：

必须输出完整操作链。

------

## 12. 禁止词总结

以下词禁止作为关键步骤：

- 创建
- 配置
- 绑定
- 修改
- 检查

除非后面跟具体操作。

------

## 13. 简洁 ≠ 省略

可以少讲理论，但不能省略步骤。

------

## 14. 模板

```markdown
目标：

前置：

步骤：
1.
2.
3.

结果：

下一步：
```

------

## 15. 核心原则总结

一句话：

**不要告诉用户做什么，要告诉用户怎么点。**

------

## 16. 优先级

1. 可执行
2. 不跳步
3. 路径清晰
4. 面向零经验
5. 再考虑简洁

------

## 17. 最终目标

消灭所有“需要用户自己猜”的步骤。
